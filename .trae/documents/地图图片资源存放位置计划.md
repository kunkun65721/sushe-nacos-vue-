# 地图图片资源存放位置计划

## 项目结构分析

根据对项目的分析，这是一个基于 SSM（Spring + SpringMVC + MyBatis）架构的 Web 项目，使用 Maven 进行构建管理。

### 当前项目静态资源配置
- **CSS 文件**：`src/main/webapp/WEB-INF/css/style.css`
- **JSP 视图**：`src/main/webapp/WEB-INF/jsp/` 目录
- **SpringMVC 配置**：使用 `<mvc:default-servlet-handler/>` 处理静态资源

## 地图图片资源存放建议

### 推荐方案：在 webapp 目录下创建 `images/maps` 文件夹

```
src/main/webapp/
├── WEB-INF/
│   ├── css/
│   ├── jsp/
│   └── web.xml
├── images/
│   └── maps/          ← 地图图片放这里
│       ├── campus_map.png
│       ├── dormitory_building_1.png
│       └── ...
└── index.jsp
```

### 具体步骤

1. **创建目录结构**
   - 在 `src/main/webapp/` 下创建 `images/maps/` 目录

2. **放置地图图片**
   - 将地图图片文件放入 `images/maps/` 目录

3. **在 JSP 中引用**
   ```jsp
   <img src="${pageContext.request.contextPath}/images/maps/campus_map.png" alt="校园地图"/>
   ```

4. **或者通过 Controller 访问**（可选）
   - 如果需要动态处理或权限控制，可以通过 Controller 来访问图片资源

## 方案优势

- ✅ **简单直接**：图片可直接通过 URL 访问，无需额外配置
- ✅ **符合规范**：符合 SpringMVC 静态资源处理惯例
- ✅ **便于管理**：集中存放，易于维护和查找
- ✅ **安全可控**：`WEB-INF` 目录外的资源可被直接访问，`WEB-INF` 内需要通过 Controller 或视图解析器访问

## 注意事项

1. 如果地图图片较大，建议进行压缩优化
2. 如果有多种类型的图片，可以考虑更细的分目录结构（如 `images/maps/`、`images/avatars/` 等）
3. 确保图片文件名使用英文或拼音，避免中文乱码问题
