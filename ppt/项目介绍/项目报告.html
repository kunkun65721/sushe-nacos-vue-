SPRING CORE

Spring IoC 与依赖注入

🎯 IoC 控制反转

将对象创建权交给Spring容器，由容器控制对象之间的依赖关系，而非传统的手动new对象。

传统方式:
UserService = new UserService();
IoC方式:
@Autowired UserService userService;

💉 DI 依赖注入

Spring容器自动将依赖的对象注入到目标bean中，支持多种注入方式。

构造器注入 推荐，不可变
Setter注入 可选依赖
字段注入 简洁但不推荐

项目中的实际应用

Service层
@Service注解
DAO层
@Repository注解
Controller层
@Controller注解

MYBATIS

MyBatis ORM 映射机制

🗂️ Mapper XML 配置

SQL语句与Java方法映射，编写原生SQL获得最大灵活性

<mapper namespace="...">
  <select id="findById"
          resultMap="StudentMap">
    SELECT * FROM student
    WHERE id = #{id}
  </select>
</mapper>

🔗 ResultMap 映射

将查询结果自动映射为Java对象，支持嵌套查询

<resultMap id="StudentMap"
         type="Student">
  <id     property="id"
          column="id"/>
  <result property="name"
          column="name"/>
</resultMap>

TRANSACTION

事务管理机制

保证数据一致性的核心机制

🔄 编程式事务

通过TransactionTemplate手动控制

transactionTemplate.execute(status -> {
  studentMapper.insert(s);
  allocationMapper.insert(a);
  return true;
});

📌 声明式事务

通过@Transactional注解简化事务控制

@Transactional(rollbackFor = Exception.class)
public void allocate(Student s, Dormitory d) {
  studentMapper.update(s);
  dormitoryMapper.update(d);
}

事务传播行为

REQUIRED
默认，有则加入
REQUIRES_NEW
创建新事务
NESTED
嵌套事务
READ_ONLY
只读事务
